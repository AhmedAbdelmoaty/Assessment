Descriptive Statistics Assessment — MCQ only (2 per level)

Always MCQ, single correct answer, 3–5 plausible choices, concise wording.

Personalize contexts/examples by sector, job nature, experience, and user profile.

Language = current UI language (clean AR/EN; keep function tokens in backticks if any).

Levels & clusters (must not ask outside these)

Level 1 — Foundations

Measurement Scales & Data Types — nominal/ordinal/interval/ratio; numeric/categorical/datetime; valid summaries per scale.

Central Tendency Basics — mean vs. median vs. mode; when each is appropriate; simple numeric interpretations.

Basic Spread & Distribution Shape — range and qualitative shape (symmetry/skew, modality); read basic histograms/frequency tables.

Level 2 — Core Applied Descriptives

Quantiles, IQR & Boxplots — quartiles/percentiles, IQR, Tukey fences; side-by-side boxplots for group comparison.

Standard Deviation & Variability Metrics — variance/SD, coefficient of variation; interpret spread relative to scale and mean.

Grouped Summaries (Categorical × Numeric) — per-group mean/median/IQR, weighted vs. unweighted summaries, pivots; side-by-side box/violin/bar charts for descriptive comparisons.

Level 3 — Professional Descriptive Skills

Z-Scores & Standardization — compare across units/scales; compute and interpret standardized values.

Skewness, Kurtosis & Q–Q Diagnostics — descriptive shape measures; read Q–Q plots; choose transformations for clearer summaries.

Correlation vs. Covariance (Bivariate) — magnitude/direction and units; read scatterplots; beware nonlinearity/heteroscedasticity.

Deterministic progression (implement exactly)

For each level (L1 → L2 → L3), generate two distinct MCQs covering different clusters inside that level (avoid asking the same cluster twice if possible).

Both correct → advance to the next level.

One correct, one wrong → advance to the next level, record the wrong MCQ’s cluster as a Gap.

Both wrong → retry with two new questions (same level, different clusters if possible) once only.
If the retry also results in both wrong, stop here. Do not go to higher levels.

If we stopped early, all clusters in higher, unvisited levels are considered Gaps.

Evaluation is local/deterministic: match the selected choice string to correct_answer. Keep per-session evidence:

evidence: [
  { level: "L1|L2|L3", cluster: "central_tendency_basics", correct: true|false },
  ...
]

Model I/O formats (strict)
MCQ JSON (model → server)
{
  "kind": "question",
  "level": "L1|L2|L3",
  "cluster": "central_tendency_basics",
  "type": "mcq",
  "prompt": "…localized prompt…",
  "choices": ["…","…","…","…"],
  "correct_answer": "…",
  "rationale": "One or two short lines explaining why the answer is correct."
}

Final report JSON (model → server)

Single opening line (≤ 2 short sentences) tailored to performance.

Strengths: concise cluster names (not long headings).

Gaps: concise cluster names.

One overall stats_level only: Beginner | Intermediate | Advanced.

No links, no sources.

{
  "kind": "final_report",
  "message": "Short opening...\nStrengths:\n- ...\nGaps:\n- ...",
  "strengths": ["central tendency basics","measurement scales"],
  "gaps": ["iqr & boxplots","z-scores"],
  "stats_level": "Intermediate"
}

server/index.js — routes & memory shape

In-memory sessions.set(sessionId, { lang, intake, level: "L1", attempts: 0, evidence: [], gaps: [], finished: false }).

Routes:

POST /api/intake/next → drives the next intake prompt; validates previous answer; returns the next message + optional choices/dropdown spec.

POST /api/assess/next → requests a new MCQ from the model for the current level+cluster window (pass profile + disallowed clusters asked in this level).

POST /api/assess/answer → checks choice vs correct_answer, updates evidence, enforces progression rules, decides next step or stop.

POST /api/report → sends the full evidence + inferred Strengths/Gaps (engine may pre-mark gaps from unvisited higher levels) to the model to produce the final_report JSON above.

Keep the system prompt in prompts/system.js and call it with parameters { lang, profile, level, allowedClusters, avoidClusters, evidence }.

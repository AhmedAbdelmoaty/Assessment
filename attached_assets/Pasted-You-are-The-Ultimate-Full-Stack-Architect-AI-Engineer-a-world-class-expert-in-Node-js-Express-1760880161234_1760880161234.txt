🔐 AUTHENTICATION FLOW
1️⃣ First Visit

User fills Intake → includes email + name.

Backend:

Creates new user (unverified).

Sends Magic Link email with token.

User clicks the link:

Backend marks email verified.

Frontend shows Set Password Form.

User sets password → login starts → chat continues to assessment.

2️⃣ Returning Visit

On load, /api/state checks session.

If authenticated → resume where left off.

If not → show Login Block inside chat area (replaces old intake messages).

3️⃣ Login Block

Fields: Email + Password + “Forgot password?”

Submit → /api/auth/login

After login → hide block → resume user’s saved progress.

4️⃣ Forgot Password

/api/auth/request-reset → sends reset email.

/api/auth/reset → verify + set new password.

⚙️ API CONTRACTS
Endpoint	Method	Description
/api/auth/start	POST	Create unverified user + send magic link
/api/auth/verify?token=	GET	Verify email token
/api/auth/set-password	POST	Set first password after verification
/api/auth/login	POST	Normal email/password login
/api/auth/logout	POST	End session
/api/auth/request-reset	POST	Send reset link
/api/auth/reset	POST	Confirm reset and set new password
/api/state	GET	Return last known state (intake/assessment/report/teach)
/api/me	GET/PUT	Get or update user profile
/api/attempts	GET	List user’s assessments
/api/attempts/new	POST	Start a new attempt
/api/attempts/:id	GET	Fetch details for one attempt

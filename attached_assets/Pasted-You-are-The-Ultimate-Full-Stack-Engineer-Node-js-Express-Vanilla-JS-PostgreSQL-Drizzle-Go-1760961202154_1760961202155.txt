BACKEND ENDPOINTS (exact contract)

Auth

POST /api/auth/signup
Body: { name, username, email, phone?, password, confirmPassword }
Validates + creates user (email_verified=false), inserts OTP record (6 digits, expires_at=now()+15m, consumed_at=null), send email if SMTP, else log:
console.log("[DEV MODE] OTP for", email, ":", code);
Response: { ok:true }
(Do NOT return the OTP in the HTTP body.)

POST /api/auth/verify-otp
Body: { email, code }
Logic: fetch latest UNCONSUMED OTP for that user, check expires_at, compare as strings, on success set consumed_at=now(), set users.email_verified=true.
Response: { ok:true }

POST /api/auth/login
Body: { email, password } → on success create session (req.session.userId) and return { ok:true }.

POST /api/auth/logout → destroy session { ok:true }.

GET /api/session/state → returns { loggedIn, stage } where stage ∈ {"intake-pending","assessment-cta","assessment-in-progress","teaching","idle"} for last known state.

Profile

GET /api/me → profile fields.

PATCH /api/me → editable fields (name, username, phone, plus intake fields stored in profileJson if you keep a merged view); enforce unique username.

Intake

POST /api/intake/next
Request identifies user by session. Do NOT ask for name/email/phone.
Returns next prompt/options or { done:true }.

Assessment

POST /api/assess/start → create or resume an assessment_sessions with status='in-progress'.

POST /api/assess/next → returns next MCQ.

POST /api/assess/answer → body { sessionId, qIndex, choiceIndex }, save answer.

When complete → compute score_percent, set finished_at, status='complete', persist Report & Teaching (see below).

Teaching

POST /api/teach/start → create teach_sessions linked to the last completed assessment.

POST /api/teach/message → append tutor responses (your current logic).

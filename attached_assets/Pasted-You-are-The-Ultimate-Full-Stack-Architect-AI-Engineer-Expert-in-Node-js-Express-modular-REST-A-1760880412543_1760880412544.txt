AUTH & SESSION FLOW
First-time (after Intake)

User finishes intake (includes email + name).

Backend creates users row (unverified), generates verify token (random 32–48 bytes), stores SHA256 in email_tokens with type='verify', expires_at=now()+15m.

Send Magic Link: GET /api/auth/verify?token=<raw> (token validated server-side by hashing & matching, not stored raw).

On verify success:

Set email_verified_at=now().

Respond with page/modal to Set Password: POST /api/auth/set-password (min length, complexity).

On success: create session, redirect to chat → assessment starts.

Returning user

On app load, call /api/state:

If session valid: return last known app state (intake/assessment/report/teach) → resume seamlessly.

Else: show Login Block (inside chat page), hide old intake messages.

Login: POST /api/auth/login (email + password).

Logout: POST /api/auth/logout.

Forgot Password

Request: POST /api/auth/request-reset (always 200; if user exists & verified, send reset link, else silent).

Reset click: GET /api/auth/verify?token=<raw>&type=reset → show Set New Password modal → POST /api/auth/reset.

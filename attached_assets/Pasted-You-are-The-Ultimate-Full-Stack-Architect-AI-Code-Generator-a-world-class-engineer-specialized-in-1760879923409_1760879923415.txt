🔐 Authentication Flow
🧭 First Visit (New User)

User completes intake (name, email, etc.)

Backend auto-creates user (unverified)

Sends Magic Link email with token

On click → verifies email → shows Set Password screen

User sets password → gets logged in → starts assessment

💬 Returning Visit

User lands on site → if not logged in:

Show login block inside chat page, replacing old intake messages.

Fields: Email + Password + “Forgot Password?”

After login → resume from last saved point.

🔄 Password Reset

“Forgot password?” sends reset link → user clicks → sets new password → login auto-restored.

🧩 Session Handling

Use PostgreSQL session store with connect-pg-simple

Expire sessions after inactivity (configurable TTL)

Secure cookies (httpOnly, secure, sameSite)

🧱 Backend API Contracts
Endpoint	Method	Description
/api/auth/start	POST	After intake, create unverified user + send Magic Link
/api/auth/verify?token=	GET	Verify email token
/api/auth/set-password	POST	Set first password after verification
/api/auth/login	POST	Email + Password login
/api/auth/logout	POST	End session
/api/auth/request-reset	POST	Send reset link
/api/auth/reset	POST	Confirm reset + set new password
/api/state	GET	Get user’s last known state
/api/me	GET/PUT	Get or update user profile
/api/attempts	GET	List assessment history
/api/attempts/new	POST	Start new assessment attempt
/api/attempts/:id	GET	Get single attempt with score details

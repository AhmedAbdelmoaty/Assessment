🐞 CURRENT PROBLEM (Diagnosis)

The file public/index.html currently combines Home + Chat into a single page.

public/app.js contains both the home logic and the chat logic together.

When the user clicks Chat from the dashboard, it actually loads the Home section, requiring them to click Start Assessment to reach Chat.

The header shown inside Chat is the Home header — not the Dashboard header.

The dashboard.html page is static (non-interactive) because no JS file is connected to handle API calls (profile info, assessments, explanations, etc).

Root Cause

Two distinct pages (Home + Chat) share the same HTML + JS file, creating state conflicts and incorrect navigation.
